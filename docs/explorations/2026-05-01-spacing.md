# Visual identity — spacing, radii, depth metaphor (session 4)

Session 4 of the multi-session visual-identity design pass (per
[`../ui/foundations/README.md → Sessions`](../ui/foundations/README.md#sessions)).
Output is the **spatial contract** — base spacing unit + scale,
component-internal padding tokens, radii vocabulary, depth
metaphor ("flat, nothing flashy" cashed out), and the **density
toggle ship-or-cut decision** (cut for v1).

This file is an exploration record. Once integrated, the canonical
home is [`../ui/foundations/spacing.md`](../ui/foundations/spacing.md).

## Decisions locked entering this session

- **Density toggle — cut for v1.** Single density posture
  calibrated for both mobile and desktop. The user clarified at
  Q1 that they had been thinking "standard + generous" rather
  than "standard + compact"; cutting to single comfortable is
  consistent with the generous lean and avoids the 2×
  visual-test footprint across every UI pass.
- **Spacing scale via Tailwind utilities.** No semantic
  `--gap-*` tokens minted. Tailwind 3's `gap-N` / `p-N` / `mt-N`
  utilities cover the long tail; minting tokens duplicates the
  scale without adding semantic value.
- **Component-internal padding tokens minted.** Six tokens for
  cross-component consistency (`--row-pad-y` / `-x`,
  `--input-pad-y` / `-x`, `--button-pad-y` / `-x`). Single
  values; no density variants.
- **Radii vocabulary — four tokens.** `--radius-sm` (4 px),
  `--radius-md` (8 px), `--radius-lg` (12 px), `--radius-full`
  (9999 px). Tailwind-aligned with a slight bump from defaults
  for a contemporary feel.
- **Radii structurally locked, not themeable.** Geometry
  identity, not theme identity. Re-addable later if real demand.
- **Depth metaphor — pure flat.** Zero shadow tokens ship.
  Surfaces communicate elevation via `--bg-overlay` color delta,
  `--border` / `--border-strong` outlines, and a fixed
  mode-dependent backdrop scrim for modals. Popovers use
  color-delta + border only (no scrim).

## Depth metaphor — pure flat

Documented in
[`spacing.md → Depth metaphor`](../ui/foundations/spacing.md#depth-metaphor).

The most distinctive design call of session 4. Three rationales
favor pure flat over flat-with-overlay-shadows:

- **Identity match** — "flat, nothing flashy" reads as zero
  shadows.
- **Cross-platform parity** — avoids RN's iOS-`shadow*` vs
  Android-`elevation` vs web-`box-shadow` divergence.
- **Theme authoring** — theme authors only tune `--bg-overlay`
  color; no shadow values to calibrate per palette.

Theme-authoring guidance: `--bg-overlay × --bg-base` should be
≥ 1.3:1 luminance ratio (soft check, folds into the existing
session 2 audit utility — no new followup).

Pure-black canvas (`--bg-base: #000000`) makes the scrim
invisible; documented as a known limitation. Theme authors avoid
pure-black canvases anyway (system fonts render poorly there).

## Adversarial-pass findings (recorded)

- **Tap-target sizes under iOS / Android minima.** Resolution:
  visible-size contract; tap zone expanded via `hitSlop` on
  native. Documented as implementation-pattern guidance in
  [`spacing.md → Tap-target on native`](../ui/foundations/spacing.md#tap-target-on-native).
  No contract change needed.
- **Scrim invisibility on pure-black canvas.** Edge case; theme
  authors avoid pure-black canvases anyway (font-rendering and
  contrast-target reasons). Documented as known limitation;
  `--scrim-bg` slot revisitable if real demand surfaces.
- **Border-only depth on close-luminance `--bg-overlay`.** Theme
  authors must perceptibly differentiate `--bg-overlay` from
  `--bg-base`. Folds into session 2 audit utility as a warning.
  No new followup.
- **Theme-author shadow burden.** Themes that want shadows ship
  their own CSS at theme-author level. Foundations doesn't ship
  the slot; revisitable in v2 if demand surfaces.
- **Cross-platform RN shadow story skipped.** Verified design
  simplification.
- **Density-cut doc cascade.** Eight files; all listed in
  integration plan. Mechanical.
- **Tokens.md user-orthogonal class survival.** The class
  survives via reader font-size scale (per `typography.md`); the
  taxonomy doesn't collapse with density's removal.
- **Radii not themeable.** Considered themeable radii; rejected.
  Geometry identity is app-level, not theme-level. Revisitable
  if real demand.
