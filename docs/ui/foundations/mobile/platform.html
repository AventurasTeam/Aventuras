Platform: iOS notch · Theme: light · Status-bar style: dark-content
What this demonstrates. Safe-area regions on iOS (notch + home indicator) and Android (gesture-mode home bar / button-mode nav bar), and status-bar style binding to the active theme mode.

Toggle Safe areas to see the reserved regions (top and bottom). Interactive elements (top bar, save bar, sheet content) sit inside the safe zones; background may extend edge-to-edge. RN's react-native-safe-area-context provides the inset values via the useSafeAreaInsets() hook.

Status-bar style. Light themes → dark-content (dark icons against light background). Dark themes → light-content (light icons against dark background). Switches on theme change via RN's StatusBar.setBarStyle.

What this does not pin. Per-screen accessibility labels and focus order — session 7 retrofits. Haptic feedback specifics — post-v1 polish.
