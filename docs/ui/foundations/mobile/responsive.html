Main

Primary canvas. Survives at every viewport tier.

This pane carries the surface's primary content — narrative stream, detail-pane edit form, dashboard view.

At phone tier, the rail and detail panes collapse out (stacked-route placeholder); main is the only visible pane.

At tablet tier, rail joins main; detail collapses to a drawer placeholder.

At desktop tier, all three panes sit side-by-side.

Detail

Tertiary pane. Visible at desktop only.

At tablet and phone tiers this content surfaces via a drawer / sheet / route — the specific pattern lands in session 3 (layout primitives).

What this demonstrates. The mechanism — a viewport toggle in the review-controls bar, container-query-driven reflow, monochrome wireframe styling. Per-screen wireframes adopt the same toggle bar at session 7+ retrofit.

What this does not pin. The collapse rule itself (which pane goes where on tablet / phone, whether detail becomes a drawer or a sheet, whether rail folds to an edge strip) — that's session 4. The drawer / sheet / route vocabulary — session 3. Touch interactions on the rows (long-press, swipe-to-dismiss) — session 5.
