Known spec gaps surfaced by this PoC

      
    
        
  • Leap day position is not in the spec's schema. Earth's Feb-gets-+1 needs the calendar to declare where the year's extra day lands. This PoC adds leapDayPosition: { tier, indexedBy, index, leapAgainst } as a proposed extension. Without it, the strict spec can't express Gregorian.
    • 
        
  • Bottom tier needs an explicit rollover. Spec wording says rollover is "when this tier resets and ticks its parent" — but the bottom tier (e.g., second) also needs a declared rollover, because its parent's recursion reads it (60 seconds per minute). Without it, conversion loops forever.
    • 
        
  • Memoization needs context-free tier detection. Spec mentions "per-year cumulative cache" as the optimization. In practice, you also need to detect "constant chains" — tiers whose size doesn't depend on partial state (Earth's hour/minute/second). Caching purely by partial fills the cache with millions of redundant entries on a long story.
    • 
        
  • Pre-declared historical era flips. Stories carry era flips as state; calendars don't. But for shareable presets like Imperial Japan and Warhammer, baking the historical accession dates into the calendar is convenient. This PoC adds eras.initialFlips: Array<{ atTuple, name }> as a non-spec convenience.
    • 
        
  • "Default to today" / fixed default UX. Calendars whose origin is far in the past (Imperial Japan from 1868) are dead at worldTime=0. PoC adds defaultToToday and defaultWorldTime flags. Spec doesn't address either.
    • 
        
  • Era arithmetic with a missing year tier. Sequential calendars have no year tier; the spec says eras should be disabled but doesn't enforce it. PoC validates at JSON save.
    • 
        
  • Resolved (folded into spec): flipMode: 'display-label' | 'elapsed-from-flip' | 'calendar-aligned' — disambiguates what resetsOnFlip actually does. Imperial Japan uses 'display-label'; Warhammer uses 'elapsed-from-flip'. See spec.md → Era flip semantics.
    • 
        
  • Resolved (folded into spec): Subdivision.skipWhen: Array<Record<string, number>> — declare days that don't participate in the cycle. Lets Shire's Mid-year's Day sit outside the week so the 364-cycling-day year locks weekdays. Conditional skipping deferred: Shire's Overlithe (skipped only in leap years) requires predicate-based skip; out of v1. Consequence: leap-year Shire dates drift weekday by 1 — flagged for users.
