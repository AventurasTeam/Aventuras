What this is

      
A calendar is an ordered stack of integer counters (tiers) plus optional non-counter decorations (sub-divisions, eras). One integer — worldTime, in base units of the bottom tier — is the spine; the active calendar's tier rules convert it bidirectionally to a tuple of values, then a Liquid template renders that tuple to a display string. Sequential time (stardates) is the one-tier degenerate case of the same primitive.


      
Demo guide — what each preset uniquely shows

      
    
        
  • Earth (Gregorian) — six-tier structured calendar. 'rule' rollover on year encodes Gregorian leap (every 4, except 100, except 400). leapDayPosition declares where the extra day lands (month 1 = Feb).
    • 
        
  • Earth (BC/AD) — same structure as Earth (Gregorian), but the displayFormat uses the era_label Liquid filter to render "44 BC" for negative years and "2024 AD" for positive ones. Demonstrates the spec rule: astronomical-reference splits like AD/BC are not eras — the boundary is at a fixed point (year 1) and counting reverses on the BC side, so they're handled in displayFormat, not via narrative-triggered era flips. eras: null; default origin near year 1 so scrubbing worldTime crosses 4 BC → 1 AD visibly.
    • 
        
  • Imperial Japan (年号) — same Gregorian structure, but eras are the centerpiece. flipMode: 'display-label' means each era flip just relabels the year (Reiwa 8 instead of 2026); month/day continue from the underlying clock. Click any era's → button to see eraYear reset to 1.
    • 
        
  • Warhammer 40K Imperial — two tiers (year + 1/1000-year fraction). flipMode: 'elapsed-from-flip' means M41 → M42 structurally resets the year tier (display goes 999 → 000) while the absolute counter (in the breakdown table) keeps climbing. The same calendar moment displays differently from Imperial Japan because of the flipMode discriminator.
    • 
        
  • Tolkien Shire Reckoning — 15-cell month table mixing 1-day intercalary holidays (2 Yule, 1 Yule) with a 3-day Lithedays block and twelve 30-day months. subdivisions[].skipWhen takes Mid-year's Day out of the weekday cycle, so 365 − 1 = 364 = 52×7 and weekdays lock year-to-year (Tolkien's perpetual-calendar design). Leap years still drift by 1 day because conditional skipping (Overlithe) is deferred.
    • 
        
  • Mayan Long Count — five-tier vigesimal counter with no months, no weeks, no eras. Demonstrates pure mixed-base structural counting: baktun → katun (20) → tun (20) → winal (18) → kin (20). The 18 in the middle is the famous quirk that makes a tun ≈ 360 days rather than aligning to a solar year. Proves the schema isn't Earth-shaped — no labeled tiers, no sub-divisions, just a number stack.
    • 
        
  • Stardate (sequential) — single tier, no rollover. Display via Liquid filters: {{ count | divide: 1000 | round: 1 }}. Demonstrates the schema isn't year/month/day-locked.
    • 
        
  • Custom (build your own) — minimal valid skeleton (year + day, no months, no week). Picking it auto-opens the JSON editor. Use the schema reference below to build out tiers / sub-divisions / eras from scratch, or copy the structure from one of the demo presets.
    • 
      


      

        Schema reference — full type shape
        
type CalendarSystem = {
  id: string                       // stable id; e.g. 'earth-gregorian'
  name: string                     // human-readable label
  baseUnit: string                 // bottom tier's name in vocabulary terms
  tiers: Tier[]                    // ordered top-down; bottom tier = base unit
  displayFormat: string            // Liquid template; tier-tuple in scope
  eras: EraDeclaration | null      // null = no era support
  leapDayPosition?: {              // PoC extension — see spec gaps
    tier: string                   //   which tier gets the leap day
    indexedBy: string              //   which tier indexes into tier
    index: number                  //   value at which leap is added
    leapAgainst: string            //   which tier's 'rule' rollover defines leap
  }
  defaultOrigin: TierTuple         // worldTime=0 anchor for the demo
  defaultToToday?: boolean         // PoC: jump to wallclock now on load
  defaultWorldTime?: number        // PoC: alternative fixed default
}

type Tier = {
  name: string                     // 'year' | 'month' | 'day' | …
  startValue: number               // typical 0 or 1
  rollover: TierRollover | null    // null only for sequential top tier
  labels?: string[]                // tier-value to label (month names)
  subdivisions?: Subdivision[]     // cycling overlays
}

type TierRollover =
  | { kind: 'constant'; value: number }
  | { kind: 'table'; indexedBy: string; values: number[] }
  | { kind: 'rule'; against: string; base: number; conditions: LeapCondition[] }

type LeapCondition = {
  every: number                    // matches when (target % every) === 0
  offset?: number                  // shift the cycle (default 0)
  exclude?: boolean                // negate the previous match
}

type Subdivision = {
  name: string                     // 'weekday'
  length: number                   // cycle length (7 for week)
  offset: number                   // value at calendar's epoch
  labels: string[]                 // length === Subdivision.length
  skipWhen?: Array  // partial tuples that opt out of cycle
}

type EraDeclaration = {
  flipMode: 'display-label' | 'elapsed-from-flip' | 'calendar-aligned'
  resetsOnFlip: string[]           // tier names; semantics depend on flipMode
  defaultStartName: string         // era at worldTime=0
  presetNames?: string[]           // canonical sequence (Meiji, Taishō, …)
  initialFlips?: Array<{           // PoC convenience — pre-loaded flips
    atTuple: TierTuple
    name: string
  }>
}

type TierTuple = Record<string, number>
