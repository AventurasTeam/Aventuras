## Cross-cutting observations

These are systemic patterns that recur across multiple findings — fixing the pattern fixes many instances.

1. **The principles/patterns split has a long tail.** When patterns/ was extracted from principles.md, URLs were updated but labels weren't (16 sites still say `[principles → X]`). Beyond labels, several patterns docs' "Used by" lists are out-of-date relative to what cites them, and several pattern-shaped sections still live in principles (Wireframe format, status pill click-to-cancel detail, banner contracts). One mass label-rename pass + a "Used by" audit + 1-2 promotion candidates would close most of the tail.

2. **Per-screen docs heavily restate cross-cutting boilerplate.** `## Top-bar`, `## Save session`, `## Detail pane — raw JSON viewer`, per-row import menu prose — the same paragraph appears in 3-5 places each. This is the failure mode the principles/patterns split was supposed to prevent. A single consolidation pass collapsing each per-screen restatement to "use the standard pattern + screen-specific deviations only" could shed substantial mass and drift risk.

3. **Wireframes lag spec on edge states.** Three high-stakes states are documented but not demonstrated: Branch navigator paused-during-generation; Story Settings definitional-change confirmation modal; Chapter Timeline disabled-during-generation. Each is a load-bearing edge case that gates a dangerous action. Reviewers can't see them today.

## What looks healthy

The audit isn't a hatchet job — substantial surfaces are clean and consistent.

- **Mechanical anchor integrity is perfect.** `pnpm lint:docs` passes; remark-validate-links resolves all ~370 anchors. No lint-config gap to file.
- **Followups format hygiene is upheld.** No `[done]` annotations, no strikethroughs, no "RESOLVED" tags. The outstanding-only invariant holds; the issues are content (resolved entries that should have been deleted), not format.
- **Bracketed inline text is clean.** No `[A|B]` / `[Foo ‖ Bar]` instances outside backticks or fenced code (which would otherwise tickle remark's reference-link parser).
- **Wireframe ↔ doc alignment is strong on most surfaces.** Onboarding, Plot panel, Rollback-confirm, World panel, the reader composer's main flow all align well — chrome, row indicators, modals, in-pane sub-headers, accordion grouping, save bars all match. The drift is concentrated in Story Settings (Memory tab segment, definitional-confirm modal) and Story-list (Import + banner + help-icon).
- **Cross-doc bidirectional citations are mostly symmetric.** `architecture.md ↔ ui/principles.md` Edit-restrictions cross-citations are bidirectional; `data-model.md → Aventuras file format` ↔ `patterns/data.md → Import counterparts` are consistent; `calendar-systems/spec.md → Resolves in followups.md` matches followups.md content (notwithstanding the resolved-not-removed finding).
- **Pattern doc structure (`patterns/README.md` + 7 pattern files) is solid.** Each pattern has a clear scope and a "Used by" list (even if some are out-of-date). The structural choice to split principles from patterns is paying off — the issue is keeping the boundary clean as new content lands, not the boundary itself.
- **Wireframe basename = doc basename rule is upheld.** Every per-screen doc has a matching `.html` colocated in the same directory. The single exception (`spec.md` next to `poc.html`) is a different artifact class (proof-of-concept, not wireframe) and is correctly named.
- **Composer modes / narration / mode / lead orthogonal-axes story is consistent.** Multiple per-screen docs handle the four-axis encoding (`mode` × `lead` × `narration` × `composerWrapPov`) without conflating any. Hard to do given the conceptual proximity; the docs do it.
- **Genuine open followups are still genuine.** Manual worldTime correction (cascade vs jump), top-K-by-salience long-term memory, granular per-story model controls, FTS5 search upgrade, image generation, encryption-at-rest — all remain real parked decisions, correctly scoped, and shouldn't be touched in any consolidation pass.
