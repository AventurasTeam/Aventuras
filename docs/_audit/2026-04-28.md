## Cross-cutting observations

These are systemic patterns that recur across multiple findings — fixing the pattern fixes many instances.

1. **The principles/patterns split has a long tail.** When patterns/ was extracted from principles.md, URLs were updated but labels weren't (16 sites still say `[principles → X]`). Beyond labels, several patterns docs' "Used by" lists are out-of-date relative to what cites them, and several pattern-shaped sections still live in principles (Wireframe format, status pill click-to-cancel detail, banner contracts). One mass label-rename pass + a "Used by" audit + 1-2 promotion candidates would close most of the tail.

2. **Per-screen docs heavily restate cross-cutting boilerplate.** `## Top-bar`, `## Save session`, `## Detail pane — raw JSON viewer`, per-row import menu prose — the same paragraph appears in 3-5 places each. This is the failure mode the principles/patterns split was supposed to prevent. A single consolidation pass collapsing each per-screen restatement to "use the standard pattern + screen-specific deviations only" could shed substantial mass and drift risk.

3. **Calendar / time-tracking shape rename trail is incomplete.** The universal-seconds + `vault_calendars` + `worldTimeOrigin: TierTuple` decisions all landed in canonical docs but their downstream propagation is half-done. Reader-composer's worldTimeOrigin is "ISO 8601"; multiple docs still reference `app_settings.calendars`; the PoC still uses `baseUnit`; data-model schema-diagram comment internally contradicts its own decision section. A single pass through every calendar-touching reference (grep for `baseUnit`, `app_settings.calendars`, `worldTimeOrigin`, `worldTime`) would close this.

4. **Followups.md has accumulated commentary.** Two entries are pure restatement of decided behavior (Non-linear narrative, Fictional calendar systems) and should be deleted. Several others (Cover display, Bulk operations, Provider deletion, Bulk select, Storybook design-rules) are duplicated against per-screen open questions or canonical doc sections — pick one home each. Two TBDs (Token budgeting, lore-management agent behavior) are in canonical docs and should move down.

5. **Wireframes lag spec on edge states.** Three high-stakes states are documented but not demonstrated: Branch navigator paused-during-generation; Story Settings definitional-change confirmation modal; Chapter Timeline disabled-during-generation. Each is a load-bearing edge case that gates a dangerous action. Reviewers can't see them today.

6. ~~**Glyph collisions in placeholder ASCII.**~~ Resolved by adfe4a7 — placeholder vocabulary picked per concern (App Settings ⚙, Story Settings ⛭, Actions ⚲, Branch ⎇) and documented in principles.md with explicit anti-reuse note for ⎇.
